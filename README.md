## Исходный код и его логика

В папке [src](./src) лежат файлы с реализацией задачи, заданной в лабораторной работе

В файле [protocols.py](./src/protocols.py) описаны (с помощью *typing.Protocol*) контракты для задач и источников задач

В файле [task_module.py](./src/task_module.py) описана предварительная реализация класса-приемника задач. Методы проверки соответствия объекта протоколам, описанные в этом классе, работают в runtime (т.к. протоколы помечены как *runtime_checkable*) и используют для проверки такие функции как hasattr(), isinstance(), type() и т.д. Отдельная проверка реализована для задач типа dict, т.к. поля словаря не считаются соответствием протоколу. Функции проверки вызываются при добавлении задач/источников задач. Также установлен логгер, отображающий статус выполненных действий в консоль/файл

В файлах [task_examples.py](./src/task_examples.py) и [task_source_examples.py](./src/task_source_examples.py) инициализированы примеры различных задач (DataClass/Class/NamedTuple) и их источников (File/Generator/API), не связанных между собой наследованием

В файле [task.py](./src/task.py) описан основной класс задач Task с проработанной инкапсуляцией и валидацией атрибутов в целях сохранения валидности состояния.

В файле [task_collections.py](./src/task_collections.py) описан класс очереди задач, позволяющий эффективно обрабатывать наборы объектов Task

### Описание атрибутов и логики класса Task: 

**Атрибуты:**

- id - уникальный идентификатор, тип str или int, обязательный. При изменении автоматически обновляет множество всех ID.
- payload - полезная нагрузка, любой тип.
- description - описание задачи, тип str, по умолчанию пустая строка.
- priority - приоритет, тип int, диапазон от 0 до 10 (задается в константах).
- status - статус задачи, тип str. Допустимые значения: created, started, finished (задается в константах). При переходе в started устанавливается таймер started_at, в finished - finished_at. При возврате в created таймеры сбрасываются.
- planned_time - запланированное время, принимает timedelta, int или float (секунды), положительное значение.
- created_at - время создания, только чтение. Устанавливается при инициализации.
- started_at - время старта, только чтение. Устанавливается при переходе в статус started.
- finished_at - время завершения, только чтение. Устанавливается при переходе в статус finished.
- duration - длительность выполнения, только чтение. В статусе created - None, в started - текущая длительность, в finished - кэшированное значение.
- is_ready - вычисляемый атрибут. True если payload не None, id не None и статус created или finished. Возможна любая перезапись (т.к. атрибут не влияет на состояние экземпляра)
- is_valid - вычисляемый атрибут. Возвращает строку с процентом заполненности и списками заполненных и пустых атрибутов. Возможна любая перезапись (т.к. атрибут не влияет на состояние экземпляра)

**Методы:**

- init(id, payload, description, priority, status, planned_time) - конструктор. id обязателен, остальные опциональны.
- delete()/del - удаление задачи, удаляет ID из множества.
- setattr - расширенное поведение для создания запрета на добавление атрибутов.
- get_all_ids() - метод класса, возвращает копию множества всех существующих ID.

**Дескрипторы:**

- Data дескрипторы (с set): PriorityAttr, PlannedTimeAttr, IdAttr, DescriptionAttr, CreatedAtAttr, StatusAttr, DurationAttr. Они перехватывают присваивание и чтение, обеспечивают валидацию и контроль доступа. CreatedAtAttr и DurationAttr запрещают запись (read-only).
- Non-data дескрипторы (без set): IsReadyAttr, IsValidAttr. Они вычисляют значения на лету при чтении. Присваивание таким атрибутам создаст обычный атрибут экземпляра, перекрывающий дескриптор.
- started_at - property, только чтение. Используется для получения времени начала выполнения задачи.
- finished_at - property, только чтение. Используется для получения времени завершения задачи.

**Особенности:**

- Множество _all_ids хранит все существующие ID, автоматически обновляется при создании, изменении и удалении задач.
- Запрещено добавление новых атрибутов через setattr, разрешены только защищенные (с _) и атрибуты из константы ALLOWED_TASK_ATTRIBUTES.
- При изменении ID старый ID удаляется из множества, новый добавляется.
- При удалении задачи через delete или del ID удаляется из множества.


### Описание логики класса TaskQueue

**Основные возможности:**

- Прямой обход: ```for task in queue```

- Обратный обход: ```for task in reversed(queue)```

- Чтение: ```task = queue[0]```,```slice = queue[1:3]```

- Изменение: ```queue[0] = new_task```

- Удаление: ```del queue[0]```

- Фильтрация:
```queue.filter_by(status="done", priority=1)``` - генератор, лениво возвращающий задачи с указанными атрибутами

- Конкатенация (+, +=):
    Поддерживаемые типы:
    1) Task - добавить одну задачу
    2) TaskQueue - объединить две очереди
    3) list[Task] - добавить список задач

- Вычитание (-, -=):
    Поддерживаемые типы:
    1) Task - удалить конкретный объект
    2) TaskQueue / list[Task] - удалить все задачи из переданной коллекции
    3) int - удалить задачу с указанным id
    4) callable - удалить задачи, удовлетворяющие условию: queue - (lambda t: t.priority < 3)

- Прочие методы:
    1) ```len(queue)``` - количество задач
    2) ```task in queue``` - проверка наличия задачи
    3) ```queue.is_empty()``` - проверка на пустоту

- Обработка ошибок:
    1) При попытке добавить или установить объект неверного типа выбрасывается TaskQueueException
    2) При конкатенации списка с не-Task объектами - исключение
    3) При вычитании неподдерживаемого типа - исключение
    4) Модификация очереди во время итерации приводит к досрочному завершению и StopIteration
